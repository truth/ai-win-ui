# H5 · Embeddable Host API

**Status:** Implemented (Wave3 H5b / H6 / H7 + parent embed) — v0.3.1  
**Date:** 2026-07-16  

## Goal

Allow a third-party Win32 process to embed `ai-win-ui` by linking a static
library and using a small public C++ API — without forking a process per window.

## Public surface

```cpp
#include <ai_win_ui/host.h>
#include <ai_win_ui/version.h>

ai_win_ui::HostCreateInfo info{};
info.layoutPath = L"layouts/core_validation.xml";
info.size = L"1000x700";
info.renderer = ai_win_ui::Backend::Direct2D;

auto host = ai_win_ui::Host::Create(hInstance, info);
return host->Run(); // or external pump + host->ProcessMessage(msg)
```

| API | Role |
|-----|------|
| `Host::Create` | DPI bootstrap + window + layout load |
| `Host::Run` | GetMessage loop until quit |
| `Host::ProcessMessage` | External pump helper |
| `Host::GetHwnd` / `IsClosed` | HWND integration |
| `HostCreateInfo::parent` | `WS_CHILD` fill parent client (v0.3.1) |
| `FitToParent` / `ResizeClient` | Respond to shell `WM_SIZE` |
| `Version()` / `VersionNumber()` | `0.3.1` / `AI_WIN_UI_VERSION` |
| `OpenSecondaryHost` | Multi-window (gallery-style) |

## Build targets

| Target | Kind | Notes |
|--------|------|--------|
| `ai_win_ui_lib` | STATIC lib | Core engine + `UiHost` |

```powershell
.\build.ps1 -Configuration Debug

```

CMake: link `ai_win_ui_lib` (alias `ai_win_ui::lib`), include `include/`.

### Child embed (v0.3.1)

```cpp
info.parent = shellHwnd;          // WS_CHILD
info.ownMessagePump = false;      // shell owns GetMessage
// on shell WM_SIZE:
host->FitToParent();
```

Sample: `samples/embed_host` creates an overlapped shell and embeds the UI as a child.

## Non-goals (still)

- Cross-process remoting  
- COM / WinRT projection  
- Full UIA  
- Layered/custom chrome **inside** a child host (forced borderless system surface)
